Задание
Контекст: встреча аналитика/менеджера с заказчиком МегаБайт по сбору требований к витрине дневных продаж. Со стороны МегаБайт — Анна Крылова (руководитель коммерции) и Дмитрий Волков (руководитель розницы).

Источник (на выбор):
• Реальная встреча вашей команды — аудио 8–35 мин, транскрипт легальным средством (Whisper локально, облако). Обезличьте участников (Роль_1, Роль_2...).
• Синтетика — сгенерируйте транскрипт встречи по требованиям к витрине МегаБайт через LLM с пометкой «синтетика». Используйте `starters/w1_proto_transcript.md`.

Продукт: протокол — деловой русский, разделы: повестка, обсуждённые темы, принятые решения, action items (ответственный ролевой ярлык + горизонт), открытые вопросы, риски/зависимости (>=2).

Роль менеджер: добавьте блок «приоритизация action items» — какие задачи блокируют следующий этап проекта.

Как распределить работу с LLM:
- 🤖 LLM: извлечение структуры из транскрипта, группировка по разделам, формулировка action items.
- ✍️ Вручную: проверить что LLM не добавил решения, которых не было на встрече, уточнить сроки и ответственных, добавить контекст который «между строк».

В README.md обязательно:
• Выбранный путь (А/Б) и обоснование
• Полный текст ≥1 ключевого промпта
• ≥3 конкретных правки после LLM с пояснением «почему»
• Краткое описание каждого файла в архиве
• Описание источника (аудио/синтетика)

В README пометьте: «Путь А: реальный (обезличен)» или «Путь Б: синтетика».

Критерии приёмки

• Транскрипт в архиве (txt/md).
• Протокол — отдельный файл, разделы заголовками.
• >=3 action items или пояснение почему нет.
• ≥2 пункта в протоколе добавлены из контекста проекта (не из транскрипта) — помечены «[контекст]».
• Указан источник: реальный/синтетика.
• Источник данных (→ score_business): свои реальные данные из проекта (обезличены) → макс. балл; собственная синтетика → стандартный балл.
Чему вы научитесь
• Использовать structured output промптинг: задать LLM шаблон протокола с фиксированными разделами и получить заполненный результат.
• Детектировать «галлюцинации-дописки» — LLM склонна добавлять решения и action items, которых не было в транскрипте; уметь сверять с оригиналом.
• Применять роль-промптинг «ты — опытный секретарь проектного комитета» для повышения качества делового языка.
• Понимать что LLM теряет контекст «между строк» — неявные договорённости и подтекст всегда требуют ручного дополнения.

Как работать с LLM:
1. Подайте на вход: полный транскрипт встречи + инструкцию «выдели разделы: повестка, обсуждённые темы, решения, action items, открытые вопросы, риски».
2. Попросите: «Составь протокол встречи в деловом русском. Для action items укажи ответственного и срок. Не добавляй информацию, которой нет в транскрипте».
3. Проверьте вручную: не «дописала» ли LLM решения, которых не было; корректны ли приписанные ответственные; нет ли потерянных тем.
4. Доработайте: добавьте контекст «между строк» (что подразумевалось, но не было сказано явно), уточните формулировки решений.
