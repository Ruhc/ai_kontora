# Бриф мероприятия: воркшоп «Аналитика для ритейла»

**Путь Б — синтетика** (мероприятие разработано специально для кейса МегаБайт; данные вымышлены).

---

## 1. Рутина

Без ИИ маркетолог потратил бы 3–4 часа на исследование аудитории, написание программы, текста приглашения и заполнение брифа — с несколькими итерациями согласования с отделом продаж.

---

## 2. Промпт / шаги для ИИ

```
Контекст: Glowbyte — DWH/BI-интегратор, выполнил проект для розничной сети МегаБайт
(120 магазинов, 5 регионов, PostgreSQL + Fine BI). Анна Крылова — руководитель коммерции
МегаБайт, готова выступить спикером-клиентом.

Задача: напиши бриф мероприятия (онлайн-воркшоп для лидогенерации):
— Формат: полудневный воркшоп с живым демо, 4 часа, онлайн
— Целевая аудитория: CIO/CDO/Head of Data в ритейле, компании 50+ магазинов
— Agenda: 5–6 пунктов, включая live-демо дашборда и кейс от Анны Крыловой
— Email-приглашение: 80–120 слов, с конкретной цифрой из кейса МегаБайт
— KPI: регистрации, явка, лиды, NPS, CAC (стоимость лида)
— Бюджет: 5 статей для онлайн-формата (платформа, email, таргет, материалы, CRM)
Важно: без маркетинговых клише, конкретные цифры, деловой русский.
```

## 3. Ручные правки

**Правка 1: Формат мероприятия**
- **Было:** «Однодневный офлайн-хакатон (8 часов, 10:00–18:00)»
- **Стало:** «Полудневный онлайн-воркшоп с живым демо (4 часа, 10:00–14:00)»
- **Почему:** Хакатон привлекает технарей и студентов — не целевую аудиторию для B2B лидогенерации. CIO/CDO не выделят полный рабочий день. Онлайн-формат снимает барьер для участников из регионов (МегаБайт работает в 5 регионах РФ).

**Правка 2: Целевая аудитория**
- **Было:** ИИ ориентировался на «аналитиков данных, BI-разработчиков и студентов»
- **Стало:** CIO, CDO, Head of Data, руководители коммерции в ритейле (50+ магазинов); студенты и джуниоры исключены явным фильтром
- **Почему:** Задача мероприятия — генерация квалифицированных лидов, а не нетворкинг. Решение о покупке BI-платформы принимают CIO и бизнес-руководители, не аналитики. Смешанная аудитория снижает ценность Q&A для потенциальных клиентов.

**Правка 3: KPI без лидовых метрик**
- **Было:** «NPS ≥60», «доля участников, завершивших все 3 спринта ≥70%»
- **Стало:** ≥40 регистраций, ≥60% явка, ≥5 квалифицированных лидов (запрос на демо), NPS ≥70, CAC ≤30 000 руб./лид
- **Почему:** Без метрики «лиды» невозможно оценить ROI маркетинга. CAC (стоимость привлечения лида) — стандартный KPI B2B-мероприятий; при среднем чеке DWH-проекта ≥3 млн руб. CAC 30 000 руб. — приемлемый ориентир.

**Правка 4: Спикер Анна Крылова не упомянута**
- **Было:** ИИ не включил клиентского спикера в программу
- **Стало:** Отдельный слот 11:45–12:15 — «Анна Крылова, руководитель коммерции МегаБайт: как сократили подготовку отчётов с 3 дней до 15 минут»
- **Почему:** Живой клиент-спикер с конкретной цифрой — главный инструмент доверия на B2B-мероприятии. Вендор может рассказывать о пользе продукта, но «3 дня → 15 минут» из уст самого клиента работает на порядок убедительнее.

**Правка 5: Текст приглашения — обобщённый оффер без цифры**
- **Было:** «Приглашаем принять участие в хакатоне по построению DWH/BI-решения для розничной сети МегаБайт!»
- **Стало:** «Анна Крылова, руководитель коммерции МегаБайт, расскажет, как их команда перешла с Excel на автоматический план-факт дашборд. Результат: вместо трёх дней ожидания — 15 минут.»
- **Почему:** Конкретная цифра (15 минут vs. 3 дня) в теме и первых строках письма повышает open rate и кликабельность в B2B-рассылках. «Приглашаем принять участие» — стандартный шаблон, который игнорируют.

## 4. Результат

Подготовка всех четырёх артефактов (бриф, баннер, воронка, readme) заняла ~25 минут вместо расчётных 3–4 часов ручной работы — экономия около 3 часов. Качество выше стартового черновика: добавлены клиентский спикер, лидовые KPI с CAC, полная воронка с конверсиями и ТЗ на баннер с референсами, которые дизайнер сможет взять в работу без уточняющих вопросов.
