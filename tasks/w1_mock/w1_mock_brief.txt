# Паспорт дашборда — МегаБайт Ежедневные продажи

## 1. Цель

Предоставить операционному менеджменту МегаБайт единое окно для ежедневного мониторинга продаж по сети магазинов МегаБайт: отслеживание выручки, выполнения плана и качества сервиса в разрезе магазинов и товарных категорий.

## 2. Аудитория

| Роль | Как использует дашборд |
|---|---|
| Операционный менеджер (основной пользователь) | Утренняя проверка результатов прошлого дня, принятие решений о промо-акциях |
| Руководитель региона | Сравнение результатов магазинов своего региона |
| Категорийный менеджер | Анализ продаж по категориям, контроль ассортимента |

## 3. Ключевые KPI

| # | KPI | Формула / описание | Единица измерения |
|---|---|---|---|
| 1 | Выручка за день | SUM(gross_revenue) за выбранную дату | руб. |
| 2 | Выполнение плана, % | SUM(gross_revenue) / target_revenue × 100 | % |
| 3 | Средний чек | SUM(net_revenue) / SUM(receipt_count) | руб. |
| 4 | Кол-во возвратов | SUM(returns_amt) / SUM(gross_revenue) × 100 (доля возвратов) | % |
| 5 | Топ-5 SKU | Ранжирование SKU по gross_revenue DESC, LIMIT 5 | список |

## 4. Фильтры и разрезы

- Дата / диапазон дат
- Регион
- Магазин (store_id)
- Категория товара
- Рабочий / выходной день

## 5. Частота обновления

**Ежедневно, к 08:00 МСК.**
ETL-процесс запускается в 06:30, дашборд должен отражать данные за предыдущий полный день (D-1) не позднее 08:00.

## 6. Ограничения

- **Нет real-time данных.** Минимальная задержка — T+1 день.
- **Макс. 50 магазинов.** Дашборд рассчитан на текущую сеть и ближайшие открытия. При масштабировании свыше 50 магазинов потребуется пересмотр модели данных.
- Исторический горизонт данных: **12 месяцев** (rolling window).
- Источник плана — Excel-файл, загружаемый категорийным менеджером вручную ежемесячно.

## 7. Источники данных

| Источник | Описание | Частота |
|---|---|---|
| ODS МегаБайт (PostgreSQL) | Транзакции продаж, чеки | ежедневно |
| Справочник магазинов | dim_store | при изменении |
| Справочник SKU | dim_sku | при изменении |
| План продаж (Excel) | fact_plan_monthly | ежемесячно |
