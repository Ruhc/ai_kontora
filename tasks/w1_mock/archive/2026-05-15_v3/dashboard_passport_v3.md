## 4. KPI — определения (без изменений v2)

См. паспорт v2.0 (`dashboard_passport.md` §3). Ключевое: **% плана MTD** считается от **ожидаемого MTD** (план × прошедшие рабочие дни / рабочие дни месяца), шкала 70 / 90 / 100%.

| KPI | Fine BI виджет | Поле |
|---|---|---|
| Выручка дня | Indicator | `net_revenue` |
| Выполнение плана MTD | Indicator + цвет | `mtd_achievement_pct` |
| Средний чек | Indicator | `avg_receipt_value` |
| Доля возвратов | Indicator | `returns_pct` |
| Магазинов с продажами | Indicator | `COUNT(DISTINCT store_id)` |

## 5. Визуализации и карта Fine BI

| # | Виджет Fine BI | Название на дашборде | Датасет / join | Примечание реализации |
|---|---|---|---|---|
| 0 | Text / Banner | T+1 SLA | — | Статический текст |
| 0b | — | Роль (RLS) | `dim_store`, `dim_sku` | Row/column permissions |
| 1–5 | Indicator | KPI (5) | `mart.daily_order_kpis` | Direct, агрегация на выбранную дату |
| 6 | Line | YoY дневная выручка | `daily_order_kpis` | Две серии + условные точки &lt;90% |
| 7 | Line | Прогноз плана MTD | `daily_order_kpis` + план | 4 серии: факт, прогноз, план месяца, ожидаемый MTD |
| 8 | Column | Сравнение недель | `daily_order_kpis` | Clustered column |
| 9 | Table | Слабые дни | `daily_order_kpis` | TOP N, фильтр achievement &lt; 90% |
| 10 | Table | Слабые магазины | `daily_order_kpis` + `dim_store` | TOP 5, achievement &lt; 90% |
| 11 | Table | Слабые категории | `fact_sales_daily` + `dim_sku` + план | achievement &lt; 70% |
| 12 | Grouped Bar | Регионы Факт vs Ожидаемый MTD | `daily_order_kpis` + `dim_store` | Цвет по светофору |
| 13 | Cross Table | Категории | `fact_sales_daily` + `dim_sku` + `fact_plan_monthly` | Строка «Нет плана» |

**Не реализуется в Fine BI нативно (макет HTML):** интерактивный narrative — в проде заменить текстовым виджетом или закладкой «Выводы».

## 8. SLA и ограничения

Без изменений относительно v2 (§7–8 паспорта v2.0):

- ETL 06:30 МСК, SLA дашборда **08:00 МСК**, данные **T+1**
- MVP рабочих дней: пн–пт
- Сеть ≤150 активных магазинов
- План из Excel; отсутствие плана — статус **«Нет плана»**, метрики % и отклонения не считаются
- Закрытые магазины исключены (`is_active = TRUE`)

## 9. Gap-анализ: макет vs прод Fine BI

| Элемент макета | Риск / действие |
|---|---|
| Линейный прогноз (серия «Прогноз») | Рассчитать в DWH view или calculated field; Fine BI Line поддерживает |
| YoY «те же даты» | View с `calendar.same_date_ly` или self-join |
| Проблемные точки на Line | Условное форматирование точек / отдельный слой |
| TOP N таблицы | Fine BI Table + сортировка + лимит строк |
| RLS демо-пилюли | Заменить на реальные роли Fine BI |
| Narrative | Ручной текст или отдельный слайд |
