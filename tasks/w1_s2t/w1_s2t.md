Задание
Вы — системный аналитик на проекте DWH для МегаБайт (розничная сеть, ~120 магазинов, 5 регионов, PostgreSQL + Power BI/Fine BI). Заказчик — Павел Чернов (CIO). S2T-маппинг — стандартный артефакт аналитика на этапе обследования источников в проектах DWH/BI. Задача: составить Source-to-Target маппинг для витрины `mart.daily_order_kpis` (зерно: день * магазин).

Обязательные колонки S2T-таблицы: target_table, target_column, data_type, source_system, source_table, source_column, transformation_rule (текстовое описание или SQL-выражение), business_rule / комментарий, is_nullable, default_value.

Объём: >=12 строк маппинга (ключи + метрики + служебные поля типа load_ts). Минимум 3 строки с нетривиальным transformation_rule (агрегация, CASE, lookup).

Роль ETL: добавьте колонку `incremental_strategy` (append / merge / SCD-2 / full) и для 2–3 полей явно опишите стратегию.

Как распределить работу с LLM:
- 🤖 LLM: генерация черновика маппинга по DDL, заполнение transformation_rule.
- ✍️ Вручную: проверить source_system/source_table (LLM выдумывает имена!), уточнить business_rule, добавить поля которые LLM пропустил (load_ts, is_deleted).

В README.md обязательно:
• Выбранный путь (А/Б) и обоснование
• Полный текст ≥1 ключевого промпта
• ≥3 конкретных правки после LLM с пояснением «почему»
• Краткое описание каждого файла в архиве
• Описание источников

Входные данные — два пути:
(А) *Реальный S2T* — из рабочего проекта с обезличиванием. (Б) *Синтетика* — используйте шаблон `starters/w1_s2t_template.md` и модель МегаБайт из задания w1_mart. В README пометьте путь.

Как работать с LLM:
1. Подайте на вход: DDL источников (stg-таблицы) + DDL целевой витрины + шаблон S2T.
2. Попросите: «Заполни S2T-маппинг: для каждого target_column укажи source, transformation_rule как SQL-выражение, business_rule».
3. Проверьте вручную: названия source_table/source_column (LLM часто выдумывает несуществующие таблицы!), корректность data_type, полноту служебных полей.
4. Доработайте: добавьте недостающие поля (load_ts, batch_id), уточните transformation_rule для сложных CASE-выражений.
