Задание
Вы — аналитик на проекте DWH для МегаБайт (розничная сеть, ~120 магазинов, 5 регионов, ~45 тыс. чеков/день, ~15 тыс. SKU, PostgreSQL + Power BI/Fine BI). Заказчики — Павел Чернов (CIO) и Дмитрий Волков (руководитель розницы). Коллеги готовят витрину продаж по дню; вам нужно собрать чек-лист из 12–16 вопросов бизнесу и ИТ прежде чем утверждать гранулярность и метрики, а затем оформить ответы как мини-BRD (бизнес-требования).

Группы вопросов (минимум по 2 в каждой):
• Гранулярность и зерно витрины;
• Определения метрик (выручка, возвраты, налоги);
• Часовые пояса и календарь;
• Качество данных источников;
• SLA обновления и формат потребления.

Дополнительный артефакт — мини-BRD (≤400 слов):
Оформите результат сбора требований как структурированный документ бизнес-требований: цель витрины, потребители и их задачи, бизнес-правила (определения метрик, исключения), ограничения, критерии приёмки (как понять, что витрина «правильная»). BRD — стандартный артефакт аналитика на этапе обследования.

Роль ETL: добавьте 2–3 вопроса про формат поставки (CDC / batch / API), идемпотентность, размер инкремента.
Роль BI: добавьте 2–3 вопроса про тип подключения (Direct Query / Import), частоту обновления дашборда, row-level security.

Как распределить работу с LLM:
- 🤖 LLM: генерация первого черновика 15–20 вопросов по группам.
- ✍️ Вручную: убрать общие/банальные вопросы, добавить вопросы из опыта, адаптировать под конкретику МегаБайт (магазины, POS, фреш).

Чему вы научитесь
• Использовать chain-of-thought промптинг: «сначала перечисли риски витрины, затем для каждого риска сформулируй вопрос» — для генерации осмысленных вопросов.
• Отличать «общие» вопросы LLM от «конкретных»: модель по умолчанию генерирует банальности — уметь фильтровать и заменять.
• Применять цикл «контекст -> черновик вопросов -> фильтрация -> дополнение из опыта» как AI-воркфлоу для любых чек-листов.
• Понимать что LLM не знает реальных источников МегаБайт — вопросы про конкретные системы (POS, ERP, WMS) всегда требуют ручной доработки.
