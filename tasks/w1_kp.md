Задание
Соберите черновик коммерческого предложения (КП) на проект внедрения хранилища данных и BI-платформы для МегаБайт (розничная сеть, ~120 магазинов, 5 регионов, ~45 тыс. чеков/день). Заказчик со стороны клиента — Ирина Соколова (COO) и Павел Чернов (CIO). Продукт — типовой проект BI-интегратора: экспресс-диагностика BI-зрелости (входная точка), независимый выбор BI-платформы по методологии 88 параметров + TCO-анализ (Fine BI / Visiology / Форсайт / Alpha BI / Superset — не привязаны к одному вендору), обследование источников, проектирование DWH, ETL-разработка, дашборды и визуализация (6-шаговый процесс: бизнес-исследование → макет → данные → прототип → тестирование → прод), обучение пользователей, опционально — Gen BI (NL2SQL для бизнес-пользователей). Объём КП — до ~600 слов или 1,5 стр. PDF.

Обязательные разделы КП (заголовками):
• Контекст и задача заказчика — боль, кто принимает решение;
• Состав поставки — функциональные границы, что вне scope;
• Ценность и экономика — TCO или эффект в условных цифрах, допущения помечены;
• Сроки и этапы — ориентир 8–12 недель, 4–6 вех;
• Риски и ограничения — >=4 пункта (данные, регуляторика, ресурсы, change mgmt);
• Коммерческая рамка — диапазон или формулировка цены + сценарий отказа.

Дополнительный артефакт — опросник для discovery-встречи (≥12 вопросов).
Перед написанием КП нужно собрать информацию у клиента. Составьте опросник с помощью LLM, сгруппированный по категориям:
• бизнес-задачи и KPI (≥3 вопроса),
• данные и источники (≥3),
• ИТ-ландшафт и ограничения (≥3),
• бюджет, сроки, команда (≥3).

Роль маркетинг: оцените, насколько КП подходит как шаблон для рассылки лидам после мероприятия; предложите 2 буллета «что бы добавили для email-рассылки».

Как распределить работу с LLM:
- 🤖 LLM: генерация структуры КП, текст разделов, формулировки ценности.
- ✍️ Вручную: проверить числа TCO/ROI (LLM выдумывает!), скорректировать сроки под реальность, убрать шаблонные фразы, добавить конкретику МегаБайт.

• >=1 измеримый критерий успеха проекта описан явно.
• Есть сценарий прекращения / отката.

Чему вы научитесь
• Использовать structured output промптинг: задать LLM жёсткую структуру (6 разделов с заголовками) и получить форматированный результат.
• Выявлять «галлюцинации в числах» — LLM генерирует правдоподобные, но выдуманные TCO/ROI; уметь помечать допущения и проверять порядок величин.
• Применять iterative refinement: первый промпт -> черновик -> «сделай конкретнее, убери воду, добавь риски» -> финальная версия.
• Понимать границу AI: LLM хорошо генерирует структуру и формулировки, но коммерческие числа и сроки — всегда ответственность человека.
