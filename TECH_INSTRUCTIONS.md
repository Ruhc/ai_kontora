# Технические инструкции для AI-ассистента

## Формат сдачи (КРИТИЧНО)

Каждая задача сдаётся через Telegram-бота командой `/submit TASK_ID`. Вложение — **один файл**: `.md` или `.zip`/`.rar`.

### Структура архива задачи
```
wN_taskid/
  readme.md          # Паспорт делегирования (ОБЯЗАТЕЛЕН)
  <артефакт_1>       # Основной результат задачи (.sql, .md, .pdf, .html и т.д.)
  <артефакт_2>       # Дополнительные материалы
  ...
```

### Паспорт делегирования (readme.md)

Каждый `readme.md` ОБЯЗАТЕЛЬНО содержит 4 раздела:

```markdown
# <Название задачи>

## 1. Рутина
<Одно предложение: что делали бы без ИИ.>

## 2. Промпт / шаги для ИИ
<Промпт или 1-2 шага, которые дали ИИ-инструменту.>

## 3. Ручные правки
<2-5 пунктов: что исправили вручную — цифры, формулировки, структура, клиентские ограничения.>

## 4. Результат
<Сэкономленное время, улучшение качества или честный провал.>
```

**Правило:** секция «Ручные правки» — самая важная для ИИ-судьи. Она показывает экспертизу. Каждая правка должна содержать: что было → что стало → почему.

## Правила генерации артефактов

### SQL (PostgreSQL 14+)
- Используй CTE, не подзапросы
- Явные типы данных в финальном SELECT (для Direct Query)
- COALESCE для обработки NULL
- Комментарии к каждому CTE (одной строкой)
- Имена таблиц с префиксом `mart.`
- Стратегия инкрементальной загрузки в комментариях

### Документы (деловой русский)
- Конкретные числа, не «значительно» / «существенно»
- Без канцеляризмов: «в рамках» → «внутри», «осуществляет» → «делает»
- Формулы и расчёты показывать явно
- Для КП/писем — не более указанного лимита слов

### Дашборды и макеты
- KPI-полоса сверху (3-5 карточек)
- Фильтры: период, регион, магазин, категория
- Цветовая индикация: зелёный (>100%), жёлтый (90-100%), оранжевый (70-90%), красный (<70%)
- Гранулярность: день → месяц → год drill-down

### Презентации
- Один ключевой тезис на слайд
- Визуалы > текст
- Структура: проблема → решение → ценность → следующие шаги

## Контексты между задачами

Задачи связаны в единый проект. При решении новой задачи:

1. **Перечитай** CLAUDE.md — там актуальный бизнес-контекст
2. **Проверь** принятые решения в таблице — не противоречь им
3. **Используй** имена стейкхолдеров, метрики, таблицы из предыдущих задач
4. **Обнови** CLAUDE.md после решения: добавь новые решения в таблицу
5. **Сохраняй** консистентность терминологии между артефактами

## Критерии оценки ИИ-судьи

На основе анализа формата хакатона, ИИ-судья вероятно оценивает:

1. **Полнота** — покрыты ли все обязательные разделы задания
2. **Качество артефакта** — профессиональный ли результат
3. **Паспорт делегирования** — насколько осмысленные правки (показывают экспертизу, а не «исправил опечатку»)
4. **Консистентность** — согласованность с контекстом проекта
5. **Бонусы** — выполнение дополнительных ролей (Аналитик, ETL, BI, Sales, Маркетинг)

## Приоритизация задач

### Стратегия максимизации баллов
- Сначала обязательные разделы, потом бонусные роли
- Лучше 6 задач на 9/10, чем 8 задач на 7/10
- Обязательно минимум 1 сдача за каждую неделю (условие для призов)
- People's Choice — отдельная номинация, стоит вкладываться в визуально впечатляющие задачи (mock, pitch, demo)

## Распределение ролей в команде (3 человека)

Рекомендуемая специализация по типам задач:

| Участник | Фокус | Типичные задачи |
|----------|-------|-----------------|
| #1 Data Engineer | SQL, ETL, dbt, DAG | mart, s2t, etlsql, etlcap, dbt, dag, batch |
